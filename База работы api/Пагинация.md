В проекте используется 2 типа пагинации, обычно она используется в List View. **Уточняйте в документации необходимой сущности какая пагинация используется**

# Постраничная пагинация

Обычная разбивка результата по страницам. 
previous и next - ссылки на предыдущую и следующую страницу если она есть.
count - количество записей соответсвующих запросу

Принимает следующие GET параметры:

- page - номер страницы
- page_size - размер страницы

Пример пагинированного результата:
```
{
	"count": 35753,
	"next": "http://example.com/api/1/app/object/?page=2",
	"previous": null,
	"results": []
}
```

# Курсорная пагинация

Пагинация сложнее в использовании, но используется в списках где не требуется работа со страницами, что увеличивает производительность.

Для следующей страницы нужно передавать get параметры курсора на последний элемент предыдущего списка (чтобы его не собирать из результирующего массива данные будут сгенерированы в next_cursor) с приставкой cursor_<cursor_key>.

**Например** мы отправили запрос на `http://example.com/api/1/app/object/` и получили такой next_cursor:
```
{
  "results": [...],
  "next_cursor": {
    "datetime": "2026-02-01T12:00:00",
    "id": 123456
  },
  "has_next": true
}
```

Для получения следующей страницы нам нужно отправить запрос на `http://example.com/api/1/app/object/?cursor_id=123456&cursor_datetime=2026-02-01T12:00:00`
